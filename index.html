分类版帮助

先看这个：一分钟判断该改哪里

遇到问题时先不要乱改。按现象判断负责区域：AI 没输出，改世界书；抓不到，改正则；显示丑，改 HTML；内容错位，看字段对应表。

AI 根本没有输出标签，或者输出不完整

问题表现：回复里没有 <状态栏>、没有某个字段，或者字段有时出现有时消失。

怎么处理：写明“每次回复结尾必须输出；字段为空也必须保留空标签；禁止 Markdown；禁止解释标签；禁止改标签名和顺序”。

问题表现：示例输出看起来有内容，但捕获结果为空。

原因：正则里的标签名、顺序、根标签、结束标签或跨行匹配与示例输出不一致。

该改哪里：正则和示例输出；如果真实 AI 输出也不稳定，再改世界书。

怎么处理：点正则区“分析整个正则每一块”；核对每个标签名是否一字不差；跨行字段用 [\s\S]*?；不要乱加普通括号。

能显示，但内容跑错位置

问题表现：地点显示成心理活动，好感显示成日记，char1 内容跑进 char2。

原因：正则捕获组顺序和 HTML 的 $1-$n 位置不一致，或者 mapping 写错。

该改哪里：字段对应表、正则捕获顺序、HTML 占位符位置。

怎么处理：先看 $1 到底是什么，再看 HTML 把 $1 放在哪里。不要只盯着颜色和样式改。

内容没错，但不好看、看不清、宽度怪

问题表现：白底黄字、按钮看不清、背景没铺满、手机端横向撑开。

原因：CSS 视觉层出问题，不是数据层出问题。

该改哪里：HTML/CSS。

怎么处理：改 color、background、opacity、width、max-width、overflow。世界书和正则通常不用动。

点击、切换、展开在网站能用，进酒馆不能用

原因：普通正则替换环境不一定执行 JS。

该改哪里：HTML 交互方案，必要时生成酒馆助手插件版本。

怎么处理：优先用 details/summary、checkbox/radio + label、CSS :checked；必须 JS 时明确要求“酒馆助手插件完整页面版本”。

一、基础关系：先搞懂三件套怎么连

这组问题解释世界书、正则、HTML、示例输出、字段对应表分别负责什么。

三件套到底怎么连起来？
世界书规定 AI 怎么输出标签；AI 原始输出是实际吐出的标签文本；正则把标签里的内容抓成 $1、$2、$3；HTML把这些 $n 放进漂亮界面。例子：世界书要求输出 <时间>晚上</时间>，正则抓到“晚上”成为 $1，HTML 里写“时间：$1”，最终显示“时间：晚上”。
世界书、正则、HTML 谁最重要？
三个都重要，但负责的事不一样。世界书管“AI 有没有输出这个字段”；正则管“能不能抓到这个字段”；HTML 管“抓到后长什么样”。所以不是哪里坏了都改 HTML。比如 AI 根本没输出“阴暗值”，你在 HTML 写再多 $9 也没用。
$1、$2、$3 是从哪里来的？
它们来自正则里的捕获组，也就是括号。第一个能抓内容的括号是 $1，第二个是 $2，以此类推。如果你在正则前面多加了一个括号，后面的 $n 可能全部顺序变化，内容就会错位。
字段对应表有什么用？
字段对应表是“翻译表”，告诉用户 $1 是时间、$2 是地点、$3 是心理活动。内容跑错位置时，先看这里。如果表里写 $4=好感，但 HTML 把 $4 放到天气位置，就要改 HTML；如果表本身就错了，要改正则或 mapping。
示例输出是不是最终效果？
不是。示例输出是 AI 原始标签文本，用来测试正则能不能抓到内容。最终效果是正则替换 HTML 后的预览。示例输出越接近酒馆里 AI 真实会吐出的内容，测试越准。

二、生成与 API：生成前后的问题

这组问题处理 API、模型、JSON、自动保存、继续修改。

自动寻找模型失败，是不是 Key 错？
不一定。可能是 Key 错，也可能是 Base URL 不支持 /models，或者网页直连被 CORS 拦截。解决：手动填模型名；检查 Base URL 是否以 /v1 结尾；换支持网页请求的中转；或以后做后端代理。
AI 返回“不可解析 JSON”怎么办？
说明 AI 没按网站要求只返回 JSON，可能在 JSON 外面写了解释文字，或者代码里有没转义的引号。解决：让 AI 重新生成，并明确说“只返回严格 JSON，不要 Markdown，不要代码块”。
为什么每次生成/修改都要自动保存？
因为 AI 改稿有时会越改越偏，自动保存能保留每个可用版本。如果新版崩了，去历史记录找上一版；这比只留当前版安全很多。
继续修改时为什么要带完整三件套？
因为用户一句“详情字改白色”只该改 HTML；但一句“新增占有欲字段”要改世界书、正则、HTML 和 mapping。AI 必须看到完整上下文，才知道该动哪几处。这也是防止它改一个地方删掉原功能。
继续修改时怎么判断“只改一块”还是“整套联动”？

只改一块：颜色、字体、背景、宽度、翻页动画、展开样式、按钮位置这类外观问题，通常只改 HTML。

整套联动：新增字段、删除字段、改标签名、改输出格式、改正则捕获组、$1-$n 错位、示例输出匹配失败，必须检查世界书、正则、HTML、示例输出和字段表。

该怎么选：停在哪个页签就以哪个页签为主；网站会根据修改意见自动判断范围。

重点：返回时仍然是完整 JSON，方便保存；但没有关系的部分会要求 AI 原样保留，不再乱重写。

AI 生成太长被截断怎么办？
调大 max_tokens，或者让它先生成简单版，再逐步增强。三件套里 HTML 最容易长；如果样式特别复杂，可以要求“保留功能，减少装饰代码”。

三、世界书问题：AI 不按格式时看这里

只要问题是“AI 没吐对标签、缺字段、乱输出说明”，优先查世界书。

AI 不按格式输出 / 标签缺失 / 有时有有时没有
优先改世界书。把“必须、严格、每次、字段为空也要输出空标签、禁止 Markdown、禁止解释标签”写清楚。如果是触发型美化，比如“查看论坛内容”才输出，要写清触发条件。
AI 把标签写成中文句子了怎么办？
世界书要给固定格式示例，并要求“严格维持标签，不要改标签名”。比如必须写 <mind>...</mind>，不要写“心理活动：……”。
AI 在标签外多写了一大段解释怎么办？
世界书里加“除标签结构外，不要输出任何额外解释、标题、Markdown、代码块”。正则也可以适当允许前后有空白，但不要靠正则硬吃所有废话。根源还是让 AI 稳定输出。
想新增字段，比如占有欲/课程表/偷藏物
三件套都要改。世界书新增标签和生成规则；正则新增捕获组；HTML 新增 $n 展示位置；mapping 新增字段说明。只改 HTML 不够，因为 AI 不知道要输出新字段。
想删除字段
也要三件套一起删。世界书删输出要求，正则删捕获组，HTML 删对应 $n，mapping 删对应行。否则 $7 可能变成 $6，后面全部错位。
双人状态栏 char1/char2 混在一起怎么办？
世界书要把 char1 和 char2 的标签层级分开，比如 <char1_status> 和 <char2_status>，不要共用一堆模糊字段。正则和 HTML 也要分两组接收，避免 char1 的 $n 放进 char2 页面。

四、正则问题：匹配失败、错位、抓不到看这里

只要问题是“有标签但预览失败、内容跑错位置、$n 对不上”，优先查正则。

预览匹配失败 / 明明有标签但抓不到
检查根标签是否一致、子标签名是否一致、顺序是否一致、结束标签有没有闭合。跨行内容建议用 [\s\S]*?。最常见：世界书写 <心理活动>，正则写 <心理>，一个字不同都抓不到。
内容错位：心理活动跑到地点、char 页跑到 user 页
看正则捕获顺序和字段对应表。第一个捕获组永远是 $1，第二个是 $2。如果正则先抓心理再抓地点，但 HTML 以为 $1 是地点，就会错位。
为什么要用 [\s\S]*?，不用 .*?
很多 JavaScript 正则里，点号默认不匹配换行。AI 输出常常有换行，所以用 [\s\S]*? 更稳。如果正则写了 /.../s，也可以让点号跨行，但为了兼容，生成器常用 [\s\S]*?。
正则里的括号可以随便加吗？
不可以。普通括号会增加捕获组，影响 $1-$n 顺序。如果只是为了分组但不想占编号，用非捕获组 (?:...)。这就是为什么正则区新增了“分析整个正则每一块”按钮。
根标签要不要写进正则？
建议写。根标签能限定这一整段属于某个美化组件，避免误抓正文里的同名标签。比如 <status_card>...</status_card> 包住所有字段，正则更稳。
正则可以只抓一大段回复区域吗？
可以。论坛体、聊天记录、楼层列表这种可变数量内容，常把整个回复区域作为一个捕获组，然后 HTML 里整体放 $3。但如果要给每条回复单独美化，普通正则替换会更复杂，可能需要酒馆助手插件或固定条数。
改了标签名后必须改哪些地方？
世界书、正则、示例输出都要同步。HTML 不一定要改，除非字段顺序或 $n 变了。改名不改正则，预览会直接匹配失败。

五、HTML 与预览问题：不好看、点不动、撑宽看这里

只要内容已经显示出来，但样式不对，主要查 HTML/CSS。

颜色丑、字看不清、背景没铺满、宽度变细长
改 HTML/CSS。文字颜色看 color，背景看 background/background-size，宽度看 width/max-width，溢出看 overflow/max-height。世界书和正则通常不用动，因为它们不管视觉。
为什么进酒馆后宽度变细长？
SillyTavern 消息楼层本身有宽度限制，HTML 里的 width:100vw 或固定大宽度可能导致缩放异常。建议最外层用 width:94%; max-width:420px; margin:auto; 手机和电脑都稳。
展开按钮点不动 / 切换页没反应
普通正则替换里的 JS 不一定执行。优先用 details/summary、checkbox/radio + label、CSS :checked。如果必须 JS，就要生成“酒馆助手插件版本”，包含完整 html/head/body。
为什么预览页能动，进酒馆不能动？
网站预览 iframe 允许更多普通网页行为，但 SillyTavern 正则替换环境可能限制脚本。所以交互组件要按酒馆环境设计，别只看浏览器预览。
HTML 里的 $1 可以删掉吗？
删掉就不会显示这个字段了。可以移动 $1 的位置，也可以在周围加标题，但别把 $1 改成中文。想隐藏字段可以不放 $n，但正则仍然可能在抓它。
背景图片不显示怎么办？
检查图片链接是否能公开访问，URL 是否完整，CSS 是否写成 background-image:url('链接')。如果被遮罩盖住，调低遮罩 opacity 或检查背景是否放在正确容器上。
手机端被撑宽怎么办？
最外层避免固定大宽度，代码块和长文本加 overflow:auto，表格加横向滚动，图片 iframe 设 max-width:100%。这类问题改 HTML/CSS，不改世界书和正则。

六、选中分析、放大查看、荧光标记

这组问题解释“AI 怎么知道你选中了哪一段”，以及怎么用荧光标记找范围。

AI 到底怎么看出我选中了哪一段？
你在文本框里拖动选中后，网页会读取 selectionStart 和 selectionEnd，也就是选区起点和终点。点击分析时，网页把选中片段、所属区域、完整三件套、字段对应表一起发给 AI。AI 不是猜，它是根据“选中片段 + 上下文”判断这段和另外两件套的关系。
为什么要加“➕ 荧光标记”？
选中文本时，手机和浏览器的原生选区不稳定，容易看不清。荧光标记会把你选中的片段复制到下面，并用荧光背景标出，方便确认范围。它不会修改原代码，只是定位预览。
分析选中段为什么还要带完整三件套？
单看一小段经常判断不了。例如 HTML 里的 $5 必须结合正则第五个捕获组和世界书对应标签，才能知道 $5 是什么。所以选中分析会带世界书、正则、HTML、示例输出和 mapping。
我选中一段后提示没有选中怎么办？
需要在代码框内部真正拖动选择文字，不是点一下。手机上可以长按文字后拖动选择范围，再点分析。如果打开了放大窗口，请在放大窗口的大文本框里重新选。
什么时候用“分析整个正则”？
当你完全看不懂正则、字段错位、匹配失败、括号太多不知道哪个是 $几时，用它。它只在正则区出现，会逐段解释每个标签、捕获组和 $1-$n 对应关系。

七、导出、导入、历史记录

这组问题处理版本保存和迁移。

导出 TXT 里应该包含什么？
应该包含 JSON 原始包、世界书、正则、HTML、示例输出、字段对应表和使用说明。这样以后导入或手动复制到酒馆都方便。
导入旧三件套后能继续改吗？
可以，只要导入内容能解析成 JSON 包。导入后可以继续预览、让 AI 修复、让 AI 根据意见改新版。如果只是散装世界书/正则/HTML，需要先整理成网站支持的 JSON 格式。
历史记录太多怎么办？
可以保留关键版本，删除不需要的版本。建议每次大改前手动保存一个“能跑版”。比如：能匹配版、样式满意版、最终导出版。

八、进阶：酒馆助手插件与复杂交互

这组问题处理 JS、iframe、triggerSlash、变量读写等复杂玩法。

什么时候需要酒馆助手插件版本？
当你需要执行 JavaScript、点击按钮触发 Slash、读取/设置变量、做复杂动态界面时，需要插件版本。普通正则替换 HTML 更适合静态展示和 CSS 交互。
插件版本和普通 HTML 片段有什么区别？
普通美化 HTML 是片段，不包含完整 html/head/body；插件版本通常要完整页面结构，并通过 iframe 执行脚本。所以生成时必须让 AI 标明“这是酒馆助手插件版本”。
triggerSlash 是做什么的？
它可以在插件 iframe 里触发 SillyTavern 的 Slash 命令，比如执行 /echo 或 Quick Reply。这属于进阶能力，不是普通状态栏必须要用的。
getVariables/setVariables 是做什么的？
它们用于读取和设置当前聊天的局部变量，可以让界面记住状态。如果只是展示状态栏，不需要它；如果要做可持久化小工具，才考虑用。
